# ⚡ Smart NILM & TNEB Cloud Ecosystem
### *Non-Intrusive Load Monitoring via Magnetic Fingerprinting & AWS Integration*

---

## 📌 Project Overview
This project implements a **Non-Intrusive Load Monitoring (NILM)** system that allows for energy auditing without the need for traditional, invasive current clamps. Developed as part of a Final Year Electrical and Electronics Engineering (EEE) curriculum, it bridges the gap between **Electromagnetic Field Theory** and **Cloud Computing**.

By utilizing the **Hall Effect Magnetometer** found in modern smartphones, the system detects minute fluctuations in the magnetic flux density ($\Delta B$) generated by current-carrying conductors. This data is processed at the edge using Python and synchronized to the **AWS Cloud** for real-time billing analytics and historical logging.

## 🚀 Key Features
*   **Non-Contact Sensing:** Uses the Biot-Savart Law to derive current from magnetic field strength, eliminating the need for electrical contact.
*   **Hybrid Data Acquisition:** Features an auto-calibration fallback system to ensure operational stability even during network latency or sensor disconnects.
*   **TNEB Billing Engine:** A custom-coded logic engine that simulates the **Tamil Nadu Electricity Board (TNEB)** bi-monthly domestic tariff slabs for accurate cost prediction.
*   **Automated Cloud Synchronization:** Seamless integration with **AWS DynamoDB** via **API Gateway** and **Lambda** for persistent data storage.
*   **Professional Reporting:** Automatically generates and dispatches a high-fidelity HTML Energy Audit report via **Gmail SMTP**.
*   **Real-time Visualization:** Instant feedback through Matplotlib-based graphical analysis of power consumption vs. cost.

## 🏗️ System Architecture
The system follows a 4-layer IoT architecture:
1.  **Perception Layer:** Smartphone Magnetometer captures raw magnetic flux ($\mu T$).
2.  **Edge Layer:** Python script performs Digital Signal Processing (DSP) and converts magnetic delta into Wattage.
3.  **Network Layer:** Data is pushed through a REST API to the AWS Cloud.
4.  **Application Layer:** Data is stored in DynamoDB and a summary is emailed to the user.

## 📖 Theoretical Background
The core calculation relies on the relationship between current ($I$) and magnetic flux density ($B$) at a distance ($r$):

$$I = \frac{B \cdot 2\pi r}{\mu_0}$$

Where:
*   $B$ is the magnetic flux density in Tesla.
*   $r$ is the radial distance from the conductor (approx. 1.5cm).
*   $\mu_0$ is the permeability of free space ($4\pi \times 10^{-7} \, T \cdot m/A$).
